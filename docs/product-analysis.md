## 1️⃣ 会议记录 → 客户跟进包

### 产品定位

> **一句话描述**: 将会议记录自动转化为客户跟进邮件、待办清单和 CRM 笔记

### 多维度分析

#### 需求分析 ✅

| 维度 | 评分 | 说明 |
|------|------|------|
| 痛点强度 | 🔴 高 | 会议后整理跟进事项是高频、耗时、易遗漏的工作 |
| 市场规模 | 🟡 中 | Freelancer + 小 Agency + 顾问 + 销售团队，约 5000 万潜在用户 |
| 付费意愿 | 🔴 高 | 直接影响收入转化，B2B 用户付费意愿强 |
| 竞争程度 | 🟡 中 | Otter.ai、Fireflies.ai 存在，但偏转录而非"跟进包" |

#### 技术可行性 ✅

| 维度 | 评分 | 说明 |
|------|------|------|
| 数据获取 | 🟢 易 | 用户粘贴文本或上传文件，无外部 API 依赖 |
| AI 处理 | 🟢 易 | 单一文本模型即可，无需微调 |
| 技术栈 | 🟢 成熟 | Next.js + Vercel Functions + R2/S3 + Postgres |
| 扩展性 | 🟢 好 | 后期可接入 Zoom/Meet API 实现自动化 |

#### 开发设计 📐

```

│                      用户界面架构                            │
```

## 3️⃣ 商品 URL → 多平台上架素材包

### 产品定位

> **一句话描述**: 输入商品 URL，自动生成多平台上架所需的文案、图片描述和营销素材

### 多维度分析

#### 需求分析 ✅

| 维度 | 评分 | 说明 |
|------|------|------|
| 痛点强度 | 🔴 高 | 跨平台上架重复劳动，电商卖家刚需 |
| 市场规模 | 🔴 高 | 全球电商卖家数千万 |
| 付费意愿 | 🔴 高 | 直接影响销售，ROI 明确 |
| 竞争程度 | 🔴 高 | 已有多个竞品，需差异化 |

#### 技术可行性 ⚠️

| 维度 | 评分 | 说明 |
|------|------|------|
| 数据获取 | 🟡 中 | 需处理反爬、结构化数据解析 |
| AI 处理 | 🟢 易 | 文本重写、格式转换 |
| 技术栈 | 🟡 中 | 需要代理池、异步任务 |
| 扩展性 | 🟡 中 | 各平台规则变化需跟进 |

#### 风险与限制 ⚠️

```
数据源风险:

┌─────────────────────────────────────────────────────────────┐
│  平台        │  难度  │  风险说明                           │
├─────────────────────────────────────────────────────────────┤
│  Shopify     │  🟢 低 │ Storefront API 需商家授权          │
│  独立站      │  🟢 低 │ Schema.org Product 结构化数据       │
│  Amazon      │  🔴 高 │ PA-API 2026.4.30 弃用，限速 1 TPS   │
│  淘宝/天猫   │  🔴 高 │ 无公开 API，反爬严格                │
│  TikTok Shop │  🟡 中 │ API 有限，需商家身份                │
└─────────────────────────────────────────────────────────────┘

建议策略:
→ MVP 只支持 Shopify + 有 Schema 的独立站
→ 明确告知用户不支持 Amazon 全平台
→ 后期通过 Chrome 插件实现"当前页面抓取"
```

| 功能 | 优先级 | 工作量 |
|------|--------|--------|
| URL 输入 + 解析 | P0 | 2 天 |
| Schema.org 数据提取 | P0 | 1.5 天 |
| 商品信息结构化 | P0 | 1 天 |
| 多平台文案生成 | P0 | 2 天 |
| Amazon bullet points | P0 | 1 天 |
| 独立站 FAQ | P0 | 0.5 天 |
| 广告短文案 | P1 | 1 天 |
| Chrome 插件 | P2 | 3 天 |

## 4️⃣ 评论/评价 → 营销落地页素材包

### 产品定位

> **一句话描述**: 收集用户评论，自动生成营销落地页素材、证言卡片和社媒文案

### 多维度分析

#### 需求分析 ✅

| 维度 | 评分 | 说明 |
|------|------|------|
| 痛点强度 | 🟡 中 | 有价值但非紧急 |
| 市场规模 | 🟡 中 | 本地商家 + 品牌 + 电商 |
| 付费意愿 | 🟡 中 | 中小商家预算有限 |
| 竞争程度 | 🟡 中 | 工具分散，无整合方案 |

#### 技术可行性 ⚠️

| 维度 | 评分 | 说明 |
|------|------|------|
| 数据获取 | 🔴 难 | 多平台 API 权限复杂 |
| AI 处理 | 🟢 易 | 文本分析、聚类、生成 |
| 技术栈 | 🟡 中 | 需处理各平台 API 差异 |
| 扩展性 | 🟡 中 | 新平台接入成本高 |

#### 数据源分析 📊

```
┌─────────────────────────────────────────────────────────────┐
│  平台        │  API 可用性  │  限制说明                       │
├─────────────────────────────────────────────────────────────┤
│  Google Maps │  🟢 可用     │ Places API，有展示政策限制      │
│  Yelp        │  🟡 部分     │ 部分 endpoint 需要 Partner 权限 │
│  Amazon      │  🔴 困难     │ 需要卖家身份 + 特殊权限          │
│  Trustpilot  │  🟡 部分     │ Business API 需企业验证         │
│  App Store   │  🔴 困难     │ 无公开评论 API                  │
└─────────────────────────────────────────────────────────────┘

MVP 策略:
→ 不做"多平台自动拉取"
→ 支持手动导入文本/CSV
→ 或只支持 Google Maps 单平台
→ 后期通过 Chrome 插件实现"当前页面导入"
```

## 🎯 综合建议

### 优先级排序

```
推荐开发顺序:

Phase 1 (立即开始):
└── 会议记录 → 客户跟进包
    ├── 原因: 数据最稳、价值最直观、开发最轻
    ├── 目标: 4-6 周 MVP 上线
    └── 验证指标: 100 付费用户

Phase 2 (Phase 1 验证后):
└── 教育文档 → 教学工作包
    ├── 原因: 技术相似度高，可复用架构
    ├── 目标: 4-6 周 MVP 上线
    └── 验证指标: 50 付费用户

Phase 3 (有稳定收入后):
└── 商品 URL → 多平台上架素材包
    ├── 原因: 收入潜力大，但技术复杂
    ├── 目标: 6-8 周 MVP 上线
    └── 限制: 先只支持 Shopify + 独立站

Phase 4 (资源充足时):
└── 评论/评价 → 营销落地页素材包
    ├── 原因: 数据源摩擦最大
    ├── 目标: 8-12 周 MVP 上线
    └── 策略: 先做手动导入，后期扩展 API
```
